    Here i want to mention a couple of tags and elements i couldnt mention in the courses earlier but they are useful. This module contains <div>, <span>, <code>, id and class elements.

  • Class And Id

    1. class is a space-separated list of the classes of the element. Classes allows CSS and JavaScript to select and access specific elements via the class selectors or functions.
    2. id defines a unique identifier which must be unique in the whole document. Its purpose is to identify the element when linking (using a fragment identifier), scripting, or styling (with CSS).

    Basially, id and class are doing the same thing but id has higher relevance then class. To explain this, we can use some simple CSS code.
    <style>
    ㅤㅤ.class1 {
    ㅤㅤcolor: red;
    ㅤㅤ}
    ㅤㅤ#id1 {
    ㅤㅤcolor: green;
    ㅤㅤ}
    </style>

    <p class="class1" id="id1"> nuclear bombs </p>

    nuclear bombs

    In the example above, we created a simple stylesheet which says: elements in class "class1" are red and the elements in id "id1" are green. The element takes both id and class parameters but id parameter stands off because it has more relevance.

  • Div

    The <div> HTML element is the generic container for flow content. It has no effect on the content or layout until styled in some way using CSS. As a "pure" container, the <div> element does not inherently represent anything. Instead, it's used to group content so it can be easily styled using the class or id attributes.
    <div class="someclass">
    Some text.
    </div>

  • Span

    <span> is very much like a <div> element, but <div> is a block-level element whereas a <span> is an inline element. But overall they are doing the same purpose.

  • Code

    The <code> HTML element displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code. By default, the content text is displayed using the user agent's default monospace font.
    <code class="someclass">
    print("Balls")
    </code>
    print("Balls")
