• Visuals In HTML

    We've looked at a lot of text so far in this course, but the web would be really boring only using text. Let's start looking at how to make the web come alive with more interesting content! This module explores how to use HTML to include multimedia in your web pages, including the different ways that images can be included, and how to embed video, audio, and even entire webpages.

  • Images

    In order to put a simple image on a web page, we use the <img> element. This is a void element (meaning, it cannot have any child content and cannot have an end tag, just like <meta> tag) that requires two attributes to be useful: src and alt. The src attribute contains a URL pointing to the image you want to embed in the page. As with the href attribute for <a> elements, the src attribute can be a relative URL or an absolute URL. Without a src attribute, an <img> element has no image to load. The alt element would save us if the image couldn't be loaded by any reason. If the image isn't loaded, it would show us the text from alt element.
    <img src="banan.jpeg" alt="Banana">
    Without any CSS it will be hard to control size and locaion of the image so don't worry if you can't set these factors at the moment.
    In the example above we use url from our storage but you can also insert the image using the absolute url(be careful using others work or you'll be punished for copyrighting).

  • Video

    Now that we are comfortable with adding simple images to a webpage, the next step is to start adding video players to your HTML. to add a video you simply need a <video> tag. This element is structured with opening and closing tag with some information between them about the video in case it can't be loaded. HTML5 supports only webm and mp4 video files.
    <video src="monkeswim.mp4" controls>

    </video>

  • Audio

    Audio works just like <video> tag but we use <audio> instead. And for audio we can use only mp3 and ogg audio files.
    <audio src="school-rooftop.mp3" controls>
    ㅤㅤ <p>Your browser doesnt support HTML5 audios.</p>
    </audio>
    Some browsers support only mp3 files and some support only ogg files so if you want to make sure everyone will get access to the content you can insert audio or video in both formats like in example below:
    <audio controls>
    ㅤㅤ<source src="audio.mp3">
    ㅤㅤ<source src="audio.ogg">
    ㅤㅤ<p>Your browser doesnt support HTML5 audios.</p>
    </audio>
    This way if browser doesnt support one of the formats it will use the other one. (This works for video files too)

  • IFrame

    By now you should really be getting the hang of embedding things into your web pages, including images, video and audio. At this point we'd like to take somewhat of a sideways step, looking at some elements that allow you to embed a wide variety of content types into your webpages the <iframe> helps embedding other web pages. For example we can add any youtube video in a small window to your website.
    <iframe ㅤㅤwidth="560"
    ㅤㅤheight="315"
    ㅤㅤsrc="https://www.youtube.com/embed/xNgW8M2s3yU"
    ㅤㅤtitle="Youtube video player"
    ㅤㅤframeborder="0"
    ㅤㅤallow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
    ㅤㅤallowfullscreen>
    </iframe>
    To get this code we can simply open any youtube video on PC, click on "Share" button and choose "Embed" option. After that you'll see the <iframe> tag with similar to the example data. Copy and paste it to your website to see following result.
    Now we can watch this youtube video in a small window on your own website and you can use <iframe> for many other options.

  • Summary

    The topic of embedding other content in web documents can quickly become very complex, so in this article, we've tried to introduce it in a simple, familiar way that will immediately seem relevant, while still hinting at some of the more advanced features of the involved technologies. To start with, you are unlikely to use embedding for much beyond including third-party content images and videos on your pages. As you become more experienced, however, you are likely to start finding more uses for them.
