• HTML Forms

    Web forms are a very powerful tool for interacting with users — most commonly they are used for collecting data from users, or allowing them to control a user interface. However, for historical and technical reasons it's not always obvious how to use them to their full potential. In the articles listed below, we'll cover all the essential aspects of Web forms and their structure.

  • Form Element

    First of all we use the <form> tag. Lets look at the example below.
    <form action="/handling-form-page" method="post">...</form>
    1. <form> Tag simply tells that we are trying to create a form.
    2. <action> attribute defines the location (URL) where the form's collected data should be sent when it is submitted.
    3. <method> attribute defines which HTTP method to send the data with (usually get or post)
    The last two elements seems pretty strange for you right now but when you'll dig deeper in the web development backend, you'll understand them.

  • Textarea, Label and Input Elements

    1. The <input> HTML element is used to create interactive controls for web-based forms in order to accept data from the user.
    2. The <textarea> HTML element represents a multi-line plain-text editing control, useful when you want to allow users to enter a sizeable amount of free-form text, for example a comment on a review or feedback form.
    3. The <label> HTML element represents a caption for an item in a user interface.
    <form action="/handling-form-page" method="post">
    ㅤㅤ<label for="name"> Your name </label>
    ㅤㅤ<input type="text" id="name" name="user_name" />
    </form>
    Since it's just an example form, submiting any data to it would give an error.


    Now lets use the <textarea> tag instead of <input> and we can submit data with multiple lines.
    <form action="/handling-form-page" method="post">
    ㅤㅤ<label for="text"> Your text </label>
    ㅤㅤ<textarea id="text" name="user_text" />
    </form>

  • Button

    The markup for our form is almost complete; we just need to add a button to allow the user to send, or "submit", their data once they have filled out the form. This is done by using the <button> element.

    The <button> element also accepts a type attribute — this accepts one of three values: submit, reset, or button.
    1. A click on a submit button (the default value) sends the form's data to the web page defined by the action attribute of the <form> element.
    2. A click on a reset button resets all the form widgets to their default value immediately. From a UX point of view, this is considered bad practice, so you should avoid using this type of button unless you really have a good reason to include one.
    3. A click on a button button does nothing! That sounds silly, but it's amazingly useful for building custom buttons — you can define their chosen functionality with JavaScript.

    Now lets see this tag in action.
    <form action="/handling-form-page" method="post">
    ㅤㅤ<label for="name"> Your name </label>
    ㅤㅤ<input type="text" id="name" name="user_name" />
    ㅤㅤ<button type="submit"> Submit </button>
    </form>

    Now we can submit the data form our form right to our server but since we don't have one this tag is completly useless. Don't give up though, programming isn't easy and you have alot to learn.
